# Budget Optimization Reference

## 1) 预算可行性分级

- **可行**：预计总花费 ≤ 预算上限的 95%
- **临界**：预计总花费在预算上限的 95%-105%
- **不可行**：预计总花费 > 预算上限的 105%

建议：
- 可行：保留核心体验并设置机动金
- 临界：先执行高收益低损耗降本动作
- 不可行：优先做结构性调整（天数/城市组合/大交通）

## 2) 降本动作优先矩阵

按“节省收益 × 体验损耗”分四象限：
- 高收益 + 低损耗：优先立即执行
- 高收益 + 高损耗：按用户偏好选择
- 低收益 + 低损耗：作为补充动作
- 低收益 + 高损耗：默认不建议

## 3) 常见降本动作库

- 大交通：时间微调、分段购买、替换出发点
- 本地交通：区域聚类减少跨区、通票替代单次购票
- 住宿：调整商圈、晚入住早出策略、分段住宿
- 餐饮：将高单价餐集中到少量重点餐次
- 体验：核心项目保留，次级项目替换为低成本同主题活动

## 4) 输出中的副作用说明模板

每个降本动作都应附加：
- 节省区间：`¥X - ¥Y`
- 体验影响：低 / 中 / 高
- 时间代价：`+/- N 分钟或小时`
- 适用条件：`人群、时段、城市特征`

## 5) 闭环回灌协议

预算方案输出后，必须给两组回灌信息：

### 给 route-planning
- 推荐交通方式变化
- 换乘复杂度变化
- 稳定性风险变化

### 给 itinerary-planning
- 建议删减/替换的活动
- 每日预算上限建议
- 节奏变化（紧凑/均衡/轻松）

## 6) 不确定性处理

若无实时价格或库存信息：
- 使用区间报价
- 标注关键假设
- 建议用户在下单前二次确认价格与退改规则
