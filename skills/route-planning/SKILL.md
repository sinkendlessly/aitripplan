# Route Planning

## Overview

该 Skill 用于把模糊出行需求转成可执行路线方案，支持单段、多段、多城市和中转场景。
输出强调“可决策”：给出主方案 + 备选方案 + 关键取舍说明，而不是只给一条路径。

## 适用触发场景

当用户出现以下表达时优先触发：
- “帮我规划从 A 到 B 的路线”
- “怎么走最省时间/最省钱/最稳妥”
- “多城市怎么顺路，不走回头路”
- “中间要经过某地/避开某地”
- “到达机场（车站）后怎么衔接下一段”

## 输入要求（若缺失需主动追问）

- **起点与终点**：地名、站点名或具体地址
- **时间约束**：出发时间、最晚到达时间、是否可夜间出行
- **成本约束**：预算上限、是否接受打车/高铁/飞机
- **偏好约束**：少换乘、少步行、避堵、舒适优先、无障碍需求
- **硬性约束**：必须经过点、禁止经过点、行李大小、同行人数

当信息不完整时，优先问“最影响决策”的 2-3 个问题，不进行无穷追问。

## 规划工作流

### Step 1: 识别任务类型

- **T1 点到点**：单起终点，目标是最优单路径
- **T2 多点串联**：多个必达点，目标是顺路与总成本最优
- **T3 跨城联动**：含城际段 + 市内接驳，目标是衔接可靠性
- **T4 时窗任务**：多个点带时间窗，目标是可行性优先

### Step 2: 明确优化目标（只设一个主目标）

可选主目标：
1. **最快到达**
2. **最低成本**
3. **最低复杂度**（少换乘、少不确定性）
4. **最高稳健性**（晚点风险最小）

如用户未说明，默认主目标为“稳健优先、时间次之”。

### Step 3: 生成候选路线

至少产出 2 条可行候选：
- **主方案**：满足主目标
- **备选方案**：在次目标上更优（例如更便宜或更稳）

每条候选都要明确：
- 分段路径（交通方式、关键站点）
- 预计时长与成本区间
- 换乘次数与关键风险点
- 触发切换条件（何时从主方案切到备选）

### Step 4: 风险与应急策略

必须覆盖：
- 关键段延误/拥堵的替代动作
- 临近发车时间阈值建议（例如提前到站分钟数）
- 兜底交通方式（最后一公里/最后一班车）

## 输出格式（强制）

按以下结构输出：

1. **结论先行（3-5 行）**
   - 推荐主方案
   - 预计总时长、总成本区间
   - 适用人群或条件

2. **方案对比表**
   - 列：方案名、总时长、总成本、换乘次数、稳定性、适用条件

3. **分段执行清单**
   - 按时间顺序给出动作：出发、换乘、到达、缓冲时间

4. **风险与备选**
   - 每个关键风险给一条可执行替代动作

5. **行动建议**
   - 当下该先做什么（如先锁票、先预约、先确认哪个点）

## 质量门槛

输出前自检：
- 是否明确主目标与次目标
- 是否至少给出 2 条候选路线
- 是否覆盖“时间、成本、复杂度、风险”四维
- 是否有可执行的应急动作（不是泛泛建议）
- 是否避免编造精确实时数据（无法确认时用区间和不确定性提示）

## 与多 Agent 协作建议（可选）

当任务复杂时按如下分工组织：
- **ManagerAgent**：收集约束、定义主目标、汇总决策
- **RouteAgent**：计算候选路径与分段衔接
- **RiskAgent**：评估延误/拥堵风险并给兜底方案
- **BudgetAgent**：比较成本并输出预算弹性建议

## 闭环协同规则（Route ↔ Itinerary ↔ Budget）

当任务涉及预算或多天游玩时，按以下规则协同：

1. **Route → Itinerary**
   - 输出“每日可达半径”和“跨区成本提示”，供行程排布减少折返。

2. **Route → Budget**
   - 输出各交通段成本区间与换乘复杂度，供预算模块识别高成本杠杆。

3. **Budget 回灌后再优化 Route**
   - 若预算要求降本，优先调整交通方式与出发时段，再重算稳健性。

4. **最终输出要求**
   - 给出“路线版本号（R1/R2）”并说明与预算、行程的差异。

## 示例触发语句

- “明早 8 点前我要从浦东到苏州工业园，行李比较多，给我稳妥路线。”
- “我这次要去杭州、南京、上海，帮我排一个不走回头路的跨城路线。”
- “同样是去机场，给我一个省钱方案和一个稳妥方案。”
