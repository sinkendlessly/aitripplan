# Route Planning Reference

## 1) 目标优先级建议

当用户未明确目标时，默认优先级：
1. 可达性与稳健性
2. 时间效率
3. 成本效率
4. 舒适性细分偏好

当用户明确目标时，执行“单主目标 + 双约束”策略：
- 主目标：时间 / 成本 / 稳定性 / 复杂度 四选一
- 约束1：预算上限
- 约束2：最晚到达时间

## 2) 候选路线评估维度

每条候选路线至少打 4 个维度：
- 时长（含缓冲）
- 成本（含潜在溢出）
- 复杂度（换乘、步行、认知负荷）
- 稳定性（受天气/拥堵/延误影响）

可采用 1-5 分制并给一句解释，避免无根据的“最优”。

## 3) 常见冲突的解决顺序

- **时间 vs 成本**：若有硬到达时限，优先时间
- **省钱 vs 稳定**：跨城或赶航班场景优先稳定
- **少换乘 vs 少花费**：老人/携大件行李优先少换乘

## 4) 风险提示模板

- 关键风险：`[风险描述]`
- 触发条件：`[何时发生]`
- 立即动作：`[可执行替代步骤]`
- 代价变化：`[时间/成本变化区间]`

## 5) 输出措辞规范

推荐：
- “预计区间”“建议预留”“若发生 X 则切换到 Y”

避免：
- “绝对最快”“100%不堵”“必然准点”

## 6) 数据不确定性处理

在无法拿到实时数据时：
- 用区间而非单点值
- 明确假设前提
- 提醒用户在执行前二次确认关键环节（班次、交通管制、营业时间）
